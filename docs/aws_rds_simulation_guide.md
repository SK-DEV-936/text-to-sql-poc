# AWS RDS MySQL Simulation Guide

The Boons Text-to-SQL agent is natively designed to swap between local execution and AWS services without any required code changes because of its clean architecture Ports and Pydantic configuration.

Currently, running `./run_demo.sh` explicitly exports `USE_IN_MEMORY_EXECUTOR=true` which instructs `boons_text_to_sql_agent/main.py` to boot up the fake SQLite engine.

To seamlessly connect your local POC directly to an AWS RDS MySQL instance (assuming you have VPN/Dev access to the VPC where RDS lives), you simply configure your `.env` flags to route the application to the `MySqlExecutor` instead.

## 1. Network Access Verification
Before starting, ensure that your local machine has network sightline to the RDS endpoint. You will likely need to:
- Be connected to your corporate AWS VPN.
- Verify the RDS Security Group allows inbound connections on port `3306` from your VPN IP.

## 2. Configure Environment Variables
Create or open the `.env` file in the root of your project directory (`boons-text-to-sql-agent/.env`). You need to explicitly disable the local executor and provide the RDS credentials.

```env
# Disable the local SQLite demo engine
USE_IN_MEMORY_EXECUTOR=false

# Point the application to your AWS RDS Endpoint
DB_HOST="your-db-identifier.cluster-xyz.us-east-1.rds.amazonaws.com"
DB_PORT=3306

# Provide the credentials (you should use a READ-ONLY user)
DB_USER="boons_readonly"
DB_PASSWORD="your-rds-password"
DB_NAME="boons"

# Keep your OpenAI key active for the LLM
OPENAI_API_KEY="sk-..."
```

## 3. Start the Application
Instead of using the `./run_demo.sh` wrapper (which hardcodes the local executor), activate your virtual environment and start the FastAPI uvicorn daemon manually using your updated `.env` settings:

```bash
# 1. Activate your python environment
source venv/bin/activate

# 2. Start the FastAPI backend
uvicorn boons_text_to_sql_agent.main:app --port 8000
```

*In a separate terminal tab, you can start the Streamlit UI to connect to it:*
```bash
source venv/bin/activate
streamlit run demo_chat.py
```

## 4. How It Works Under The Hood
When `USE_IN_MEMORY_EXECUTOR=false`, the `main.py` routing logic instantiates the `MySqlExecutor` class (located in `infrastructure/db/mysql_executor.py`). 

This class utilizes the asynchronous `aiomysql` driver (already installed in your `pyproject.toml`) to open non-blocking connections to your AWS RDS instance. Because the LLM strictly generates MySQL dialect (`LangChainTextToSqlAdapter`), all queries generated by the agent will execute flawlessly against the real RDS schema.
